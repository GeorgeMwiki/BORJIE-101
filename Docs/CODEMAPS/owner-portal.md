# Owner-Portal Codemap

**Last Updated:** 2026-05-22
**Module:** `apps/owner-portal/`
**Port:** 3001
**Tier scope:** owner (org-admin + owner-advisor personas)

## Purpose

Vite + React SPA for the property owner ‚Äî the financial decision-
maker and approver. Owns financials (billing, statements, payouts,
damage deductions), compliance (data requests, documents, settings),
analytics (growth, usage, exports), communications (broadcasts,
campaigns, templates), governance (approvals, audit log, delegation
matrix, classroom), and configuration (8+ settings panels).

## Entry points

- `src/main.tsx` ‚Äî root mount with providers.
- `src/App.tsx` ‚Äî router + provider tree.
- `src/pages/` ‚Äî 80+ top-level page components
  (e.g. `DashboardPage.tsx`, `BillingPage.tsx`,
  `ApprovalsPage.tsx`, `AnalyticsExportsPage.tsx`,
  `CommunicationsBroadcastsPage.tsx`,
  `ComplianceDataRequestsPage.tsx`, `Classroom.tsx`,
  `DelegationMatrix.tsx`, `DesktopReview.tsx`).
- `src/features/`, `src/components/`, `src/hooks/`, `src/contexts/`,
  `src/lib/`, `src/index.css`.

## Internal structure

- `pages/` ‚Äî flat list (legacy) of feature pages; ~80 entries.
- `features/` ‚Äî feature-sliced modules (newer pattern).
- `components/` ‚Äî shared UI atoms.
- `contexts/` ‚Äî auth, query client, brain, locale, currency,
  feature flags.
- `hooks/` ‚Äî useCurrency, useTenantContext, useApprovals, etc.
- `lib/` ‚Äî api-client wrappers + helpers (135 callsites threaded for
  tenant currency in Wave 4 I1).

## Dependencies

- Upstream: api-gateway (REST + SSE).
- Downstream: `packages/chat-ui`, `packages/dynamic-sections`,
  `packages/design-system`, `packages/api-client`,
  `packages/api-sdk`, `packages/genui`.

## Common workflows

- **Approve a payout** ‚Üí `ApprovalsPage` ‚Üí four-eye approval ladder
  (suggest ‚Üí recommend ‚Üí act-with-approval ‚Üí act-autonomous).
- **Review monthly statement** ‚Üí `DashboardPage` /
  `AnalyticsExportsPage` ‚Üí renders owner_statements row (PDF
  generated by payments-ledger D2 writer).
- **Configure delegation** ‚Üí `DelegationMatrix` ‚Üí writes per-action
  delegation grants honoured by api-gateway authz-policy.
- **Broadcast comms** ‚Üí `CommunicationsBroadcastsPage` ‚Üí fanout via
  notifications service.
- **GDPR / PDPA data request** ‚Üí `ComplianceDataRequestsPage` ‚Üí
  invokes the 5 GDPR self-service endpoints (Wave 3 Z3).
- **Multi-currency display** ‚Üí every money rendering uses
  `formatCurrency(amount, tenant.currencyCode)`; never hard-code
  `KES`.

## Anti-patterns to avoid

- Never hard-code `KES` ‚Äî read tenant currency context (the 135-
  callsite thread is canonical).
- Never call api-gateway with absolute base URL ‚Äî go through
  `api-client` so auth + retries + observability apply.
- New pages must use `features/` slicing ‚Äî `pages/` is legacy and
  frozen; add only critical hotfixes there.
- Long lists must be virtualised (Wave CL-8 perf work).
- Never duplicate dynamic-section layout logic ‚Äî consume
  `SectionMount`.
